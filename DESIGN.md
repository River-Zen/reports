# 教程站升级设计

## 目标

把现有单篇 Harness 教程升级为可持续维护的三篇教程站：

1. Harness 从零搭建到正式接管。
2. 小墨、阿珂、三眼的三 Agent 人控协作闭环。
3. 道德经宇宙从数据、3D 星海到内容层的完整实战。

## 现状

- 仓库：`https://github.com/River-Zen/reports`
- 分支：`main`
- 工作树：开始任务时干净。
- 已有页面：`Harness搭建教程.html`、`Harness体系-20260708.html`。
- 旧教程仍以 `执行流转层/当前状态.md` 和双 Agent 为主，新体系已迁移到 `Harness/当前状态.md` 与三 Agent 协作。
- GitHub CLI 当前未登录；部署阶段优先使用现有 Git 凭据推送，再验证 Pages。

## 页面结构

- `index.html`：教程首页与学习路径。
- `harness.html`：Harness 从 0 到正式接管。
- `three-agents.html`：三 Agent 协作闭环。
- `tao-universe.html`：道德经宇宙实战。
- `assets/site.css`：共享视觉系统和响应式样式。
- `assets/site.js`：导航、目录、高亮与复制按钮。

旧页面保留，不删除、不覆盖，避免已有链接失效。

## 质量标准

- 每篇教程说明“做什么、为什么、怎么做、如何验证、出错怎么办”。
- 代码示例可直接复制，路径使用通用占位符，不泄露本机隐私或凭据。
- 纯静态 HTML/CSS/JS，无构建依赖，适配 GitHub Pages。
- 支持键盘导航、减少动态效果偏好、移动端阅读。

## 事实源摘要

### Harness

- Harness 只管“谁、何时、做什么、做到哪、谁审查”，不存项目代码和长期知识。
- `Harness/当前状态.md` 是唯一活动事实源；旧 `执行流转层/当前状态.md` 仅保留兼容快照。
- 正式接管通过 `HARNESS-SMOKE-001` 验证：三 Agent 入口、校验脚本、恢复脚本、新旧状态哈希均通过。

### 三 Agent

- 小墨：分析、方案、复盘；阿珂：代码、文件、验证、部署；三眼：视觉只读审查。
- 任何执行都必须经过马哥审批；三眼只由马哥手动触发，其报告不是执行命令。
- 任务按视觉、代码、纯知识、审查四类分派，状态与证据统一落到 Harness。

### 道德经宇宙

- Phase 1：81 章结构化数据 + Three.js 静态双星域。
- Phase 2：搜索、筛选、悬停、GSAP 穿梭和阅读面板。
- Phase 2 重构：5000 颗三层背景星，让相机进入星海内部。
- Phase 2.5：椭圆轨道、拖尾、双主星、域色温和阅读卡性能修复。
- Phase 3：81 + 33 篇、四层内容、庄子第三星域、共享进度补间。
- Phase 3.2：中央核心 → 三主星 → 114 篇副星的三层轨道秩序，桌面和手机性能均通过。
